<task type="auto">
  <name>Task 1: Rewrite WordService — typed data, remove unnecessary Observable</name>
  <files>
    src/app/word.service.ts
  </files>
  <action>
    WordService currently wraps a static object in `of()` to return `Observable<any>`. This is pointless — the data never changes, is synchronous, and is not fetched from anywhere.

    Options:
    1. Export the data as a typed constant (no service needed)
    2. Keep the service but return the typed data directly as a signal or plain getter

    Preferred approach (keep the service for DI consistency, but fix it):
    ```typescript
    interface WordSection { [pattern: string]: string[] }
    interface WordData { [section: string]: WordSection }

    @Injectable({ providedIn: 'root' })
    export class WordService {
      private readonly _allWords: WordData = { ... };

      getSection(sectionName: string): WordSection | undefined {
        return this._allWords[sectionName];
      }

      getAllSections(): WordData {
        return this._allWords;
      }
    }
    ```

    The consumer (NextStepWordStudyComponent) currently subscribes to `this.wordService.getAllWords()` via an Observable pipeline. In Phase 4 (plan 04-03) the Word Study component will be fully rewritten. For now, if removing the Observable return breaks the consumer, provide a backward-compatible method:
    ```typescript
    /** @deprecated Use getSection() directly. Removed in Phase 4. */
    getAllWords(): Observable<WordData> {
      return of(this._allWords);
    }
    ```

    But prefer just fixing the consumer now if it's simple. The consumer does:
    ```typescript
    this.wordGroups$ = this.route.params.pipe(
      mergeMap(({ sectionName }) => this.wordService.getAllWords().pipe(
        map(value => value[sectionName])
      ))
    );
    ```
    This could become:
    ```typescript
    this.wordGroups$ = this.route.params.pipe(
      map(({ sectionName }) => this.wordService.getSection(sectionName))
    );
    ```
    Much simpler. No need for mergeMap/switchMap when the data is synchronous.

    Key rules:
    - Properly type the word data (not `any`)
    - Remove the empty constructor (already handled in 02-02 but verify)
    - Remove the Observable wrapper if the consumer can be trivially updated
    - Remove `of` import if no longer needed
  </action>
  <verify>

    grep -n ": any" src/app/word.service.ts returns zero results.
    pnpm build passes.
  </verify>
  <done>WordService properly typed, no unnecessary Observable wrapping, no `any` types. Build passes.</done>

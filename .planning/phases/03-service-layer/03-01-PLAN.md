<task type="auto">
  <name>Task 1: Rewrite AuthService async methods to use defer()</name>
  <files>
    src/app/core/auth/auth.service.ts
  </files>
  <action>
    The current AuthService has methods like `signIn()`, `signUp()`, `signOut()`, `confirmSignUp()`, `resetPassword()` that are all `async` methods using `await` on Amplify SDK calls and then manually calling `.set()` on signals.

    Rewrite pattern for mutation methods (signIn, signUp, signOut, etc.):
    ```typescript
    signIn(email: string, password: string): Observable<void> {
      return defer(() => amplifySignIn({ username: email, password })).pipe(
        switchMap(() => this.loadProfile()),
        tap(() => this.router.navigate(['/dashboard'])),
        catchError(err => {
          // handle error, update error signal
          return EMPTY; // or throwError
        })
      );
    }
    ```

    For profile loading (read operation), use `rxResource` pattern:
    ```typescript
    readonly userProfile = rxResource({
      stream: () => defer(() => getCurrentUser()).pipe(
        switchMap(user => defer(() => fetchUserAttributes(user))),
        map(attrs => this.mapToProfile(attrs)),
        catchError(() => of(null))
      )
    });
    ```

    Key rules:
    - `defer(() => amplifyCall())` — NEVER `from()`. defer() is lazy, from() is eager.
    - Mutation methods return `Observable<void>` (or Observable<result>). Components subscribe or use async pipe.
    - Read operations use `rxResource` with defer() inside the stream function.
    - Error state uses signals: `readonly error = signal<string | null>(null)`.
    - Loading state comes from rxResource (it provides `.isLoading()` automatically).
    - Remove ALL `async` keywords from methods.
    - Remove ALL `await` expressions.
    - Remove ALL try/catch blocks that manually stuff signals.

    The auth guard should continue to work — it reads `userProfile()` signal which rxResource will populate.

    Import defer from 'rxjs': `import { defer, EMPTY, of } from 'rxjs';`
    Import rxResource from Angular: `import { rxResource } from '@angular/core/rxjs-interop';`

    Also update the auth guard if it currently calls async methods — it should work with the signal-based profile from rxResource.
  </action>

  <name>Task 2: Update auth guard and auth components to consume the new Observable-based AuthService</name>

  <action>
    The auth components currently call `await this.auth.signIn(...)` etc. Since AuthService now returns Observables:

    For components with form submit handlers:
    - The submit method should subscribe to the Observable or use a pattern like:
      ```typescript
      onSubmit(): void {
        this.loading.set(true);
        this.auth.signIn(email, password).pipe(
          finalize(() => this.loading.set(false))
        ).subscribe({
          error: (err) => this.error.set(err.message)
        });
      }
      ```
    - OR use rxResource if the form submission is a resource-like pattern (unlikely for auth mutations).
    - RXJS-SIGNALS-GUIDE.md says components should read signals, not subscribe. For mutations (signIn, signUp), subscribing in the component's event handler is acceptable — the guide's "no subscribe" rule is about data loading, not fire-and-forget mutations. Use `subscribe()` for mutations only, and clean up via DestroyRef or takeUntilDestroyed if the Observable isn't one-shot.
    - Actually: Amplify auth calls are one-shot (complete after single emission). So subscribe without cleanup is fine.

    For auth guard:
    - If it currently uses `await this.auth.someMethod()`, convert to Observable-based check using the rxResource signal.
    - The guard should check `this.auth.userProfile.value()` or similar signal.
    - If the profile hasn't loaded yet, the guard may need to wait — use `toObservable(this.auth.userProfile.value)` and filter for non-undefined.

    Remove all `async` from component methods that called auth service methods.

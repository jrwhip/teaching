<task type="auto">
  <name>Task 2: Replace all for loops with functional methods</name>
  <files>
    src/app/practice/math/number-line/number-line.component.ts (for loop in markers computed),
    src/app/practice/math/inequality/inequality.component.ts (for loop in ticks computed),
    src/app/practice/math/coordinate-grid/coordinate-grid.component.ts (for loop in gridLines computed),
    src/app/practice/math/results/teacher-results.component.ts (for...of loops over classrooms/enrollments/studentIds),
    src/app/practice/math/results/parent-results.component.ts (for...of loop over links)
  </files>
  <action>
    Replace every `for` loop with functional alternatives:

    **number-line.component.ts markers computed:**
    Replace the `for (let i = this.start; i <= this.end; i++)` with:
    ```
    Array.from({ length: this.end - this.start + 1 }, (_, i) => {
      const value = this.start + i;
      return { value, leftPercent: ((value - this.start) / (this.end - this.start)) * 100 };
    });
    ```

    **inequality.component.ts ticks computed:**
    Same pattern — `Array.from({ length: this.tickCount }, (_, i) => ({ index: i, value: i - 10 }))`.

    **coordinate-grid.component.ts gridLines computed:**
    Same pattern — `Array.from({ length: 2 * this.axisRange + 1 }, (_, i) => { const value = i - this.axisRange; ... })`.

    **teacher-results.component.ts:**
    The `for...of` loops iterate over classrooms to collect enrollments, then over studentIds to collect counters. Replace with:
    - `classrooms.flatMap(...)` or `Promise.all(classrooms.map(...))` for the classroom→enrollment mapping
    - `Promise.all([...studentIds].map(...))` for the studentId→counter mapping
    Note: These are in async methods with Amplify calls. The for...of with await is sequential. Replacing with Promise.all + map is both functional AND more efficient (parallel fetches). This is the correct approach.

    **parent-results.component.ts:**
    Same pattern as teacher-results — replace `for...of` with `Promise.all(links.map(...))`.

    Run: `grep -rn "for (\|for(" src/app --include="*.ts"` to find any missed for loops.
    Also: `grep -rn "for (const\|for (let" src/app --include="*.ts"` for for...of and for...in.
  </action>
  <verify>
    grep -rn "^\s*for\s*(" src/app --include="*.ts" returns zero results.
    grep -rn "for (const\|for (let" src/app --include="*.ts" returns zero results.
    pnpm build passes.
  </verify>
  <done>Zero for loops (for, for...of, for...in) in any source file. All iteration uses functional methods. Build passes.</done>

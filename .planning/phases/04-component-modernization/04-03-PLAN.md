  <action>
    The current component has these violations:
    1. `mergeMap` instead of `switchMap` (route params should cancel previous)
    2. `tap()` used for state mutation (`tap(_ => this.words = [])`)
    3. `console.log` left in production code
    4. Mutable state (`words!: string[]`) mixed with Observable (`wordGroups$`)
    5. `suffleWords` — misspelled method name
    6. Duplicate shuffle logic in `suffleWords()` and `randomize()`
    7. Fisher-Yates shuffle is implemented wrong (should swap with random index from i to end, not 0 to i)
    8. `export class` instead of `export default class`

    Full rewrite:

    ```typescript
    @Component({
      changeDetection: ChangeDetectionStrategy.OnPush,
      imports: [WordStudyMenuComponent, WordListComponent],
      templateUrl: './next-step-word-study.component.html',
      styleUrls: ['./next-step-word-study.component.scss']
    })
    export default class NextStepWordStudyComponent {
      private readonly route = inject(ActivatedRoute);
      private readonly wordService = inject(WordService);

      // Reactive route parameter
      private readonly sectionName = toSignal(
        this.route.params.pipe(map(p => p['sectionName'] as string)),
        { initialValue: '' }
      );

      // Word groups derived from section name
      readonly wordGroups = computed(() => {
        const section = this.sectionName();
        return section ? this.wordService.getSection(section) : undefined;
      });

      // Selected words for practice
      readonly words = signal<string[]>([]);

      addWords(newWords: string[]): void {
        this.words.update(current => this.shuffle([...current, ...newWords]));
      }

      clearWords(): void {
        this.words.set([]);
      }

      randomize(): void {
        this.words.update(current => this.shuffle([...current]));
      }

      private shuffle(arr: string[]): string[] {
        // Correct Fisher-Yates (Durstenfeld) shuffle
        return arr.reduceRight((shuffled, _, i) => {
          const j = Math.floor(Math.random() * (i + 1));
          [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
          return shuffled;
        }, [...arr]);
      }
    }
    ```

    Key changes:
    - `toSignal()` for route params → signal
    - `computed()` for derived word groups (sync, since WordService.getSection() is sync)
    - No Observable pipeline at all — the data is synchronous
    - Single `shuffle()` method (correct Fisher-Yates)
    - Method names fixed: `addWords` instead of `suffleWords`
    - `export default class` for lazy-loaded route
    - Zero `tap()`, zero `mergeMap`, zero `console.log`
    - Pure signal-based state

    Update the template to:
    - Use `@if` / `@for` instead of `*ngIf` / `*ngFor` (if not already done in Phase 2)
    - Read signals: `words()`, `wordGroups()`
    - Call methods: `addWords()`, `clearWords()`, `randomize()`

    Also update the child components (word-study-menu, word-list) if their @Input/@Output contracts change. The menu emits word selections — make sure the event binding matches the new method names.

<task type="auto">
  <name>Task 1: Rewrite MathComponent class to signals and modern patterns</name>
  <files>
    src/app/math/math.component.ts
  </files>
  <action>
    Full rewrite of the component class:

    1. **State → Signals:** Convert all plain properties to signals:
       - `score`, `streak`, `highScore`, `nthValue`, `roundingValues`, `randomNumber`, `userAnswer`, `isCorrect`, `ag-Grid rowData` — all should be signals
       - Computed signals for derived state (accuracy percentage, display strings)

    2. **ngOnInit → Constructor or field initializers:**
       - Move initialization logic to constructor or use field initializers
       - localStorage reads: use a helper function or service, not direct access in the component
       - If localStorage is used for streaks/high scores, consider moving to MathResultsService (DynamoDB) for consistency. If localStorage is intentional for offline support, keep but extract to a service.

    3. **Reactive form for answer input:**
       - Replace FormsModule + ngModel with a FormControl
       - Or use signal + event binding pattern

    4. **ag-Grid integration:**
       - ag-Grid 35 works with signals. Use `signal<RowData[]>` for row data and pass to grid via `[rowData]="rowData()"`

    5. **Remove CommonModule:** Should already be done in Phase 2, but verify

    6. **Results recording:** Use MathResultsService (rewritten in Phase 3) to record attempts instead of localStorage

    7. **Remove the NthPlainTextPipe dependency** if it can be replaced with a computed signal or inline logic. If the pipe is still useful, keep it but ensure it's guide-compliant (should already be after Phase 2).

    The component should follow the same patterns as the quiz components (QuizComponent, Quiz2Component, Quiz3Component) which already use signals.

    Do NOT change the rounding problem generation logic — only modernize the Angular patterns.
  </action>
  <verify>
    grep -n "ngOnInit\|OnInit" src/app/math/math.component.ts returns zero results.
    grep -n "ngModel\|FormsModule" src/app/math/math.component.ts returns zero results.

  </verify>
  <done>MathComponent uses signals for all state, reactive forms, no ngOnInit, no CommonModule. Build passes.</done>

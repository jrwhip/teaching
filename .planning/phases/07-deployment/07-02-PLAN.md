  <action>
    Using the domain provided at the checkpoint:

    1. Add custom domain to Amplify: `aws amplify create-domain-association --app-id <id> --domain-name <domain> --sub-domain-settings '[{"prefix":"","branchName":"main"}]'`
    2. Amplify will provide DNS records (CNAME or ALIAS) that need to be configured at the DNS provider.
    3. If DNS is Route 53: Amplify can auto-configure. Otherwise, provide the DNS records for Jerry to add at his DNS provider.
    4. Wait for SSL certificate verification and domain activation.

    If DNS is managed outside AWS, this task will output the required DNS records and the next checkpoint will verify they were added.
  </action>
  <verify>
    aws amplify get-domain-association --app-id <id> --domain-name <domain> shows AVAILABLE status (or PENDING_VERIFICATION if DNS records need manual setup).
  </verify>
